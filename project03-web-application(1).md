# Project 3: Web Application

**Weight:** 35% of final grade
**Duration:** Weeks 13-16
**Emphasis:** Full integration, polish, deployment

---

## Overview

Your capstone project is a substantial web application that integrates all course capabilities: skills, MCP servers, and the Ralph Wiggum autonomous loop. This is your showcase piece - it should be something you're proud to share and that classmates can actually use.

## Project Scope

This should be **more than a simple CRUD app**. Think:
- Multiple related features
- Thoughtful user experience
- Production-ready deployment
- Something genuinely useful

## Project Ideas

### Tier 1: Full-Stack Web Apps

| Project | Description | Stack Suggestion |
|---------|-------------|------------------|
| **Link Shortener** | URL shortening with analytics | Next.js + Supabase |
| **Bookmark Manager** | Save, tag, and organize bookmarks | React + Express + PostgreSQL |
| **Poll/Survey Tool** | Create polls, share, view results | Next.js + SQLite |
| **Code Snippet Manager** | Save and organize code snippets | Vue + Firebase |
| **Dashboard** | Personal metrics dashboard | React + any API |

### Tier 2: Specialized Tools

| Project | Description | Stack Suggestion |
|---------|-------------|------------------|
| **Note-taking App** | Markdown notes with organization | Electron/Tauri + React |
| **File Organizer** | Desktop file management | Tauri + Vue |
| **Clipboard Manager** | History and organization | Electron + React |
| **Pomodoro Dashboard** | Timer + task integration | Next.js |
| **Recipe Manager** | Store and organize recipes | React + Supabase |

### Tier 3: Creative Options

| Project | Description | Stack Suggestion |
|---------|-------------|------------------|
| **Portfolio Site** | Interactive personal site | Next.js/Astro |
| **Event Countdown** | Track multiple events | React + localStorage |
| **Habit Tracker** | Daily tracking with streaks | Next.js + Supabase |
| **Reading List** | Track books with notes | Vue + Firebase |

## Required Features

Your application must have:

1. **User interface** - Clean, usable frontend
2. **Data persistence** - Information saved and retrieved
3. **Multiple features** - Not just one thing
4. **Deployment** - Live and accessible
5. **Error handling** - Graceful failure states

## Technical Requirements

### Advanced Claude Code Features

Use at least one:
- **Skills** - Create a project skill for domain expertise
- **MCP server** - Connect to external data/services
- **Ralph Wiggum** - Use autonomous loop for at least one feature

### Git Workflow

- Meaningful commit history
- Feature branches
- Issue tracking recommended

### Code Quality

- Clean frontend/backend separation
- Consistent coding style
- Proper error handling
- Environment variables for secrets

### Deployment

Your app must be deployed and publicly accessible:
- **Vercel** - Great for Next.js
- **Netlify** - Great for static + serverless
- **Railway** - Full-stack apps
- **Fly.io** - Container-based
- **Supabase** - Backend as a service

## Deliverables

1. **Working application** - Deployed and functional
2. **Git repository** - Clean history
3. **Documentation**:
   - README with full setup instructions
   - Architecture overview
   - API documentation (if applicable)
4. **Demo** - Present to class in Week 16 (10-15 minutes)

## Demo Requirements

Your 10-15 minute demo should include:

1. **Live demo** - Show the app working
2. **Architecture overview** - Explain your technical decisions
3. **Git history** - Show your development workflow
4. **AI collaboration** - Discuss how you used Claude Code
5. **Lessons learned** - What worked, what didn't

## Tips for Success

1. **Start with deployment** - Get something live on day 1
2. **Iterate publicly** - Deploy early and often
3. **Use Skills wisely** - Create a skill for your domain
4. **Try Ralph Wiggum** - Even for one feature, the experience is valuable
5. **Focus on UX** - A polished simple app beats a messy complex one
6. **Test with real users** - Have classmates try your app
